# autoevolve — автономное совершенствование стратегии DatsSol

## Описание

Автономный цикл эволюции бота. Claude итеративно модифицирует стратегию, прогоняет турнир в симуляции, оценивает результат и решает — оставить или откатить. Работает бесконечно без вмешательства человека.

## Настройка

1. **Создать ветку**: `git checkout -b autoevolve/<tag>` (тег = дата, напр. `apr17`)
2. **Прочитать файлы**: полный контекст проекта:
   - `strategy/bots/current.py` — **единственный файл который ты модифицируешь**
   - `strategy/core/rules.py` — движок симуляции (READ ONLY)
   - `strategy/core/state.py` — модель мира (READ ONLY)
   - `strategy/core/mapgen.py` — генерация карт (READ ONLY)
   - `strategy/base.py` — интерфейс бота (READ ONLY)
   - `api/models.py` — типы данных (READ ONLY)
3. **Инициализировать results.tsv** с заголовком
4. **Запустить первый турнир** — это baseline

## Правила

### Что МОЖНО менять
- `strategy/bots/current.py` — **только этот файл**. Всё fair game: алгоритмы, эвристики, скоринг, приоритеты апгрейдов, логика стройки/ремонта/диверсий, планирование маршрутов, защита ЦУ.

### Что НЕЛЬЗЯ менять
- `strategy/core/rules.py` — движок симуляции (фиксированные правила)
- `strategy/core/state.py` — модель мира
- `strategy/core/mapgen.py` — генерация карт
- `strategy/base.py` — интерфейс BaseStrategy
- `api/models.py` — типы данных
- `strategy/runner.py`, `strategy/tournament.py`, `strategy/analyzer.py` — инфраструктура

### Метрика
**Цель: максимизировать средний score за турнир из 20 сидов.**

Запуск турнира:
```bash
uv run python -m strategy.tournament --seeds 20 > run.log 2>&1
```

Извлечение результата:
```bash
grep "^current" run.log | awk '{print $2}'
```

Или из CSV:
```bash
python3 -c "
import csv
rows = list(csv.DictReader(open('strategy/experiments/runs.csv')))
scores = [float(r['score']) for r in rows if r['bot']=='current']
print(f'mean={sum(scores)/len(scores):.0f} min={min(scores):.0f} max={max(scores):.0f} n={len(scores)}')
"
```

### Критерий простоты
При равных результатах — проще лучше. Маленькое улучшение ценой уродливого кода не стоит. Удаление кода с тем же результатом — отличный исход.

## Логирование

Файл `results.tsv` (tab-separated, НЕ запятые):

```
commit	mean_score	min_score	max_score	status	description
```

- commit: short hash (7 символов)
- mean_score: средний score по 20 сидам
- min_score: минимальный score
- max_score: максимальный score
- status: `keep`, `discard`, `crash`
- description: что пробовали

Пример:
```
commit	mean_score	min_score	max_score	status	description
a1b2c3d	56914	30901	86684	keep	baseline
b2c3d4e	62100	35200	91000	keep	prioritize reinforced cells in expansion
c3d4e5f	54000	28000	82000	discard	aggressive sabotage (worse than baseline)
d4e5f6g	0	0	0	crash	OOM from caching all paths
```

## Цикл экспериментов

LOOP FOREVER:

1. Посмотри текущее состояние: ветка, коммит, последние результаты
2. Сформулируй гипотезу: что изменить и почему это должно улучшить score
3. Модифицируй `strategy/bots/current.py`
4. `git commit -am "описание изменения"`
5. Очисти предыдущие результаты: `rm -f strategy/experiments/runs.csv`
6. Запусти турнир: `uv run python -m strategy.tournament --seeds 20 > run.log 2>&1`
7. Прочитай результаты: `tail -5 run.log`
8. Если вывод пустой/ошибка — `tail -50 run.log` для диагностики
9. Запиши в results.tsv
10. Если mean_score улучшился — **keep**, оставляем коммит
11. Если хуже или равно — **discard**, `git reset --hard HEAD~1`
12. Перейди к шагу 1

### Обработка ошибок
- Краш от бага (опечатка, отсутствующий импорт) — исправь и перезапусти
- Краш от фундаментальной проблемы — откати, запиши crash, двигайся дальше
- Турнир дольше 3 минут — что-то не так, убей и откати

### НИКОГДА НЕ ОСТАНАВЛИВАЙСЯ
После старта цикла НЕ спрашивай "продолжить?". Человек может спать. Работай автономно до ручной остановки. Если кончились идеи — перечитай правила игры, комбинируй предыдущие near-miss подходы, пробуй радикальные изменения архитектуры бота.

## Правила игры DatsSol (справка)

### Ключевые механики
- 600 ходов/раунд, 1 сек/ход
- Плантации связаны ортогонально с ЦУ (диагональ не считается)
- Каждая плантация: 1 действие/ход + автотерраформация
- Действия: строить, ремонтировать, диверсия, атака бобров
- Потеря ЦУ = всё уничтожено + штраф 5% очков
- Усиленные клетки (x%7==0 && y%7==0): 1.5x очков
- Плантация исчезает при 100% терраформации

### Параметры
- MHP=50, TS=5, CS=5, RS=5, DS=10, AR=2, SR=3
- Стройка: 50 HP до завершения, CS=5/ход → 10 ходов
- Терраформация: +5%/ход → 20 ходов до 100%
- Очки: 10 за 1% (обычная), 15 за 1% (усиленная)
- Лимит плантаций: 30 (+1 за апгрейд)
- Апгрейд каждые 30 ходов, макс 15

### Congestion (перегрузка)
Каждая следующая команда через одну выходную точку: CS/RS/SE/BE уменьшается на 1

### Апгрейды
- repair_power: RS +1 (макс 3)
- max_hp: MHP +10 (макс 5)
- settlement_limit: лимит +1 (макс 10)
- signal_range: SR +1 (без макс)
- vision_range: VR +2 (макс 5)
- decay_mitigation: DS -2 (макс 3)
- earthquake_mitigation: урон -2 (макс 3)
- beaver_damage_mitigation: урон -2 (макс 5)

### Формула очков
- Обычная клетка: 20 ходов × 5% × 10 = 1000 очков
- Усиленная клетка: 20 ходов × 5% × 15 = 1500 очков
- Теоретический максимум 1 плантации за раунд = 1000-1500 очков
- 30 плантаций постоянно работающих ≈ 30000-45000 очков/раунд (без учёта конвейера)

## Идеи для экспериментов (стартовый список)

1. **Конвейер стройки**: начинать строить новую до завершения терраформации текущей
2. **Параллельная стройка**: 2-3 плантации строят одну цель (5+4+3=12 CS, 5 ходов)
3. **Маршрут к усиленным**: BFS/A* путь от спавна к ближайшим (x%7, y%7)
4. **Ранний settlement_limit**: все первые очки в лимит, расти быстрее
5. **Решётка вместо змейки**: строить 2D сеть, а не линейную цепочку
6. **Предсказание 100% терраформации**: переносить ЦУ заранее, не в последний момент
7. **Фокус на кластеры усиленных**: строить вокруг перекрёстков (x%7, y%7)
8. **Динамический приоритет апгрейдов**: менять приоритет в зависимости от фазы игры
9. **Экономия действий**: не чинить плантации которые скоро дотерраформируют
10. **Максимизация idle плантаций**: каждая плантация всегда должна что-то делать
